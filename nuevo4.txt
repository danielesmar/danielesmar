El sistema se llama Riesgo Operativo, es una migración de un sistema actual, pero se requiere rehacer todo de cero, incluyendo migrar catalogos de una base de datos de sql a una base de datos en Oracle, por lo que se requieren tareas para tal fin, entre los catalogos estan: Empleados, Clasificación, Sub clasificación, Sucursales, Unidad de Ocurrencia, Monedas, Empresas, Tipo de evento, Tipo de perdida, Linea de negocio, Unidad Ejecutora, Roles y Permisos, entre otros.

El sitema se tiene que realizar en oracle apex, por lo que se requiere una tarea para el area de arquitectura o infraestructura para que prepare los ambientes y accesos.

Todos los formatos del sistema deben ser: Fecha: dd/MM/yyyy, los formatos montos: separador de miles: coma, y separador de decimales: punto, con una presición de dos decimales, las tasas de cambio son con precisión de 4 decimales.

El sistema debe permitir crear roles, estos deben tener asociados permisos en el sistema para perfiles: Auditor, Adminsitrador, Operador, en dependencia del rol podra editar, visualizar, crear diferentes registros.

Al realizar el sistema de cero, se requiere crear la funcionalidad Administrativa de crear usuarios, editar, dar de baja,  solamente deben acceder los administradores, el usuario se crea automaticamente con el siguiente formato:

la inicial del primer nombre mas el apellido, mas un consecutivo, en el caso que ya exista el nombre se debe agregar un consecutivo, hasta encontrar uno disponible, por ejemplo:

Juan Lopez Perez: jlopez001
Juan Lopez Sanchez: jlopez002

Los usuarios seran creados a demanda, a pesar de haberse migrado todos los empleados.

 la contraseña deberá enviarse al correo previamente migrado, la pantalla de login debera permitir elegir la empresa a la cual pertenece el usuario, el sistema permite filtrar por multiples Empresas, todos los catalogos y configuraciones se deben tener presente la Empresa, un usuario administrador puede acceder a cualquier empresa siempre y cuando al momento de crearse se le configure las empresas, debe tener al menos una empresa por defecto. los usuarios operadores solamente pueden tener una empresa asociada.

Funcionalidad o epica Login, el usuario al ingresar al sistema debe solicitar usuario, contraseña y empresa, al ingresar la sesión debe durar 1hora, luego de esto debe cerrarse la sesión, y solcitar login nuvamente.

Funcionalidad de Olvidó contrasñea, el usuario en el formulario login debe aparecer un link o boton que le permita ingresar el usuario y correo, si el correo es el mismo que el registrado para el usuario en la base de datos se le permitira  regenerar una nueva contraseña temporal que le llegara al correo y debera cambiar al ingresar al login

Funcionalidad cambio de contraseña: Al ingresar al login si la contraseña usada es temporal debera redirigir a la funcionalidad cambio de contraseña, la cual debe solicitar la nueva contraseña y repetir nuevamente la contraseña una vez cambiada la contraseña podrá ingresar al sistema normalmente.

Funcionalidad salir del sistema: al dar clic debera cerrar las sesiones y redirigir a la pantalla de login.

El sistema esta compuesto de las siguientes funcionalidades: Eveentos, Revisión, Historial, Segumiento, Indicadores, Matriz, Vencidos, Adminsitración, Salir del Sistema.


Funcionalidad Registro de Evento:

Esta funcionalidad estará disponible tanto para Operadores como Administradores, con algunas restricciones.
Es una pantalla con formato Wizard, y el wizard esta compuesto por los pasos:

Registro Datos Generales, Información Descrita, Personas Involucradas, Datos Economicos, Registro de perdida

Paso uno: Registrar los datos generales se deben solicitar los siguientes campos:
Fecha ocurrencia: tipo fecha (corresponde a la fecha en que se presentó el evento el usuario podra elegir a partir del calendario o bien digitarlo), Fecha de detección: tipo fecha (Debe ser posterior o igual a la fecha de ocurrencia el usuario podrá elegir del calendario o digitarla manualmente), Tipo de perdida: catalogo (el usuario puede elegir entre los valores: Econocmica o No economica), tipo de evento: catalogo, clasificación: catalogo, sub clasificación: catalogo, unidad de ocurrencia: catalogo, area de evento: campo de texto abierto requerido.

Paso dos: Información descrita: Se requiere documentar el evento.
Los campos solcitados son los siguientes, son campos abierto y requeridos:
Descripción del evento, Documentación revisada - Procedimientos efectuados, Causa: (Motivo, razón u origen que permitió que se produjera el evento), Tema control (Detallar controles, politicas, procedimientos, normativas que han sido incumplidas), Medidas correctivas (actividades realizadas para mitigar el impacto negativo del evento), Medidas preventivas (actividades encaminadas en prevenir la ocurrencia de inconvenientes similares incluyendo el/las areas involucradas en la implementación)

Paso tres: Personas involucradas: Aqui hay tres secciones: Personas involucradas, personas detectaron, personas externas.
En cada sección existe un botón para agregar personas.

Las primeras dos son internas, es decir se debe elegir de la lista de empleados, en una nueva ventana se debe poder filtrar por: Nombre, Unidad, Cargo, permitirá elegir mas de un registro, a la pantalla principal tanto para personas involucradas, como personas detectaron.

Para las personas externas se debe mostrar un formulario que solicite: Nombre de la persona, relación de la persona y observaciones de la persona. de texto libre, en nombre de persona solamente debe permitir caracteres alfanumericos y acentos del alfabeto español eñes, etc.

Se deben ir agregando en la sección personas involucradas, tanto para Personas involucradas como personas detectaron, con la siguiente información: Colaborador, Unidad, Cargo,Estado y un botón para poder removerlo en el caso que haya seleccionado uno incorrecto.

Para las personas externas se debe mostrar la siguiente información: Nombre, Relación, Observación y un botón para elminarlo.


Paso 3: Datos economicos, si el evento es de Tipo de Perdida:  Econocmica (seleccionado en el paso 1) se debe mostrar y completar este paso, solicitando la siguiente información: Monto expuesto, monto recuperación, tipo de recuperación, moneda: tipo lista valores (Dolares, Lempiras, Euros), fecha conversión, se debe mostrar de manera lectura la siguiente información: Tasa de conversión, Monto expuesto en dolares, recuperación en dolares, y perdida neta dolar, estos ultimos campos son calculados en dependencia del monto expuesto, monto de recuperación y moneda seleccionada, deben reacccionar a los cambios de estos campos y recalcularse.

Paso 4: Registro de perdida, este paso solamente se debe mostrar si en el paso 1 se eleigió tipo de perdida: Economica  y en el paso 3 el monto expuesto no fuer recuperado o bien fue recuperado parcialmente, esta sección solamente estará disponible siempre y cuando el usuario que lo registre sea un administrador, y debe solicitar la siguiente información:

Unidad: Catalogo de lista de valores, cuenta: Catalogo de lista de valores: permite buscar y filtrar el numero de cuenta, y monto en USD, permitiendo agregar los detalles en un listado que mostrará: Unidad, Cuenta, Monto Lempira, Monto U$ y un botón para poder eliminar el registro en caso de que se haya equivocado el usuario.

Una vez completado los pasos anteriores el asistente debe mostrar una pantalla de solo lectura toda la información recompilada en una pantalla de confirmación mostrando los botones en la parte inferior atras y guardar.

Una vez guardada se debe mostrar un mensaje de exito o un mensaje de error en pantalla.
